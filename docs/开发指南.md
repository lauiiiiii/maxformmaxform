# TrustForm 开发指南

基于源码状态更新时间：2026-03-25

本文仅以当前仓库实际代码为准，适用于 `maxformmaxform` 这一套前后端工程，不再沿用旧版文档中的 MongoDB / ClickHouse 双库方案描述。

## 1. 当前技术栈

- 前端：Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router + ECharts
- 后端：Node.js 18+ + Express 4 + Knex 3
- 数据库：MySQL
- 文件上传：Multer，本地磁盘存储，后端通过 `/uploads` 提供静态访问
- 鉴权：JWT
- 导出：ExcelJS + zip 附件打包
- 测试：
  - 后端：Node.js 内置 `node:test`
  - 联调：`scripts/system-smoke.mjs`
  - 前端：`vue-tsc --noEmit` + `vite build`
  - E2E：预留 Playwright 入口，当前暂无实际用例

## 3. 环境要求

- Node.js 18 或更高版本
- npm 9 或更高版本
- MySQL 8.x

建议本地保留以下端口：

- 前端开发服务：`63000`
- 后端 API：`63002`
- MySQL：默认按 `.env.example` 使用 `3309`

## 4. 环境变量

后端读取 `backend/.env`，当前示例变量如下：

```env
DB_HOST=127.0.0.1
DB_PORT=3309
DB_USER=root
DB_PASSWORD=123456
DB_NAME=survey_system

JWT_SECRET=please-change-me
JWT_EXPIRES_IN=7d

FRONTEND_URL=http://127.0.0.1:63000
PORT=63002
UPLOAD_PENDING_TTL_HOURS=24
```

补充说明：

- 上传大小上限由 `backend/src/config/index.js` 控制，当前默认单文件 10MB。
- `UPLOAD_PENDING_TTL_HOURS` 控制未提交上传附件的清理窗口。
- 生产环境必须替换 `JWT_SECRET`。

## 7. 数据模型概览

当前 `backend/src/db/migrate.js` 会确保以下核心表存在：

- `roles`
- `depts`
- `positions`
- `users`
- `surveys`
- `answers`
- `files`
- `folders`
- `messages`
- `audit_logs`

其中：

- `surveys.questions` 为 JSON 字段，存储问卷题目定义
- `answers.answers_data` 为 JSON 字段，存储答卷内容
- `files` 同时承担后台文件与问卷上传附件记录
- `surveys` 支持 `folder_id`、`deleted_at`、`deleted_by`
- `files` 支持 `survey_id`、`question_order`、`submission_token`、`public_token`、`answer_id`

## 9. 问卷与答卷主链路

### 9.1 问卷

核心路由在 `backend/src/routes/surveys.js`：

- 创建、更新、发布、关闭
- 回收站、恢复、彻底删除
- 文件夹移动
- 公开读取已发布问卷
- 上传题公开上传
- 提交答卷
- 查询结果统计

### 9.2 答卷

核心路由在 `backend/src/routes/answers.js`：

- 分页查询答卷
- 查询答卷数量
- 查看单份答卷
- 批量删除
- 导出问卷答卷 Excel
- 打包下载问卷附件 zip

### 9.3 提交校验

`backend/src/utils/questionSchema.js` 当前已覆盖：

- 发布前题目合法性校验
- 必填校验
- 单选 / 多选 / 排序选项合法性校验
- `slider / rating / scale` 数值校验
- `matrix` 行列结构与答案结构校验
- `upload` 文件数量、大小、类型校验

## 10. 文件上传机制

上传相关实现位于：

- `backend/src/routes/files.js`
- `backend/src/routes/surveys.js`
- `backend/src/utils/uploadStorage.js`
- `backend/src/models/File.js`

当前分两类：

### 10.1 管理后台普通上传

- `POST /api/files/upload`
- `POST /api/files/upload/image`

要求登录。

### 10.2 问卷公开上传

- `POST /api/surveys/:id/uploads`

特点：

- 允许已发布问卷的公开附件上传
- 可绑定 `questionId + submissionToken`
- 返回 `uploadToken`
- 答卷提交阶段的最小文件引用是 `id + uploadToken`
- `name / url / size / type` 由服务端按文件记录解析并补齐到最终答案
- 真正提交答卷时再由后端二次校验并绑定到 `answers`
- 未提交的临时附件会按 TTL 清理

## 13. 常见开发约定

### 13.1 题型改动

新增或调整题型时，至少同步检查：

- `shared/questionTypeRegistry.js`
- `shared/questionModel.js`
- `backend/src/utils/questionSchema.js`
- `frontend/src/utils/questionTypeRegistry.ts`
- `frontend/src/views/survey/CreateSurveyPage.vue`
- `frontend/src/views/survey/FillSurveyPage.vue`
- `frontend/src/views/survey/SurveyAnswersPanel.vue`

### 13.2 结果统计改动

题目级统计逻辑当前集中在：

- `shared/questionModel.js`
- `backend/src/routes/surveys.js`
- `frontend/src/views/survey/SurveyAnswersPanel.vue`

### 13.3 不要再按旧架构开发

以下认知已经过期，不应继续作为开发前提：

- MongoDB 是主库
- ClickHouse 承担当前正式结果统计
- 统计接口只有总提交数
- `positions` 仍是空壳接口
- 附件批量下载尚未落地

## 14. 当前已知缺口

- `Playwright` 入口仍无实际用例文件
- 结果页地域统计依赖答卷来源地数据，当前数据库主链路尚未稳定落地省市采集
- 高级分析 / SPSS 相关按钮仍主要是界面占位
- `vite build` 仍有大 chunk 警告
- 历史文档中仍混有旧架构描述，继续开发时要优先看源码

## 15. 建议的开发顺序

1. 继续把题型注册、提交校验、结果统计收口到共享模型。
2. 为登录、创建问卷、公开填答、结果页、答卷下载补第一批 Playwright 用例。
3. 明确答卷来源地采集方案，再补齐地域统计正向链路。
4. 在前端拆包优化后，再补一轮构建与加载回归。
