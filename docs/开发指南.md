# TrustForm 开发指南

基于源码状态更新时间：`2026-04-01`

本文仅以当前仓库实际代码为准，适用于 `maxformmaxform` 这一套前后端工程，不再沿用旧版文档中的 MongoDB / ClickHouse 双库方案描述。

## 1. 当前技术栈

- 前端：Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router + ECharts
- 后端：Node.js 18+ + Express 4 + Knex 3 + MySQL
- 文件上传：Multer，本地磁盘存储，后端通过 `/uploads` 提供静态访问
- 鉴权：JWT
- 导出：ExcelJS + zip 附件打包
- 测试：
  - 仓库根：`npm test` / `npm run test:smoke` / `npm run test:all`
  - 后端：Node.js 内置 `node:test`
  - 联调：`scripts/system-smoke.mjs` 与 `frontend` 下的 smoke npm 入口
  - 前端：`vue-tsc --noEmit` + `vite build`
  - E2E：Playwright

## 3. 环境要求

- Node.js 18 或更高版本
- npm 9 或更高版本
- MySQL 8.x

建议本地保留以下端口：

- 前端开发服务：`63000`
- 后端 API：`63002`
- smoke 端口：默认动态分配（未设置 `PORT` 时由系统分配空闲端口）
- MySQL：默认按 `.env.example` 使用 `3309`

## 4. 环境变量

后端读取 `backend/.env`，当前示例变量如下：

```env
DB_HOST=127.0.0.1
DB_PORT=3309
DB_USER=root
DB_PASSWORD=123456
DB_NAME=survey_system

JWT_SECRET=please-change-me
JWT_EXPIRES_IN=7d

FRONTEND_URL=http://127.0.0.1:63000
PORT=63002
UPLOAD_PENDING_TTL_HOURS=24
```

补充说明：

- 上传大小上限由 `backend/src/config/index.js` 控制，当前默认单文件 10MB。
- `UPLOAD_PENDING_TTL_HOURS` 控制未提交上传附件的清理窗口。
- 生产环境必须替换 `JWT_SECRET`。

当前权限模型：

- `/api/users`、`/api/roles`、`/api/audits`：只有管理员可访问，部分限制在 service 层完成
- `/api/depts`、`/api/positions`：列表对登录用户开放，增删改要求管理员
- `/api/flows`、`/api/repos`：路由层要求登录，service 层实际要求管理员
- `/api/files`：登录后可访问；管理员可按 `uploader_id` 查询，删除允许管理员或文件所有者
- `/api/folders`、`/api/messages`：登录用户按自己的 `actor.sub` 访问
- 管理写操作会在同一事务边界内完成主数据、审计日志与操作者消息写入

## 10. 文件上传机制

上传相关实现位于：

- `backend/src/routes/files.js`
- `backend/src/routes/surveys.js`
- `backend/src/utils/uploadStorage.js`
- `backend/src/models/File.js`

当前分两类：

### 10.1 管理后台普通上传

- `GET /api/files`
- `POST /api/files/upload`
- `POST /api/files/upload/image`
- `DELETE /api/files/:id`

要求登录。

### 10.2 问卷公开上传

- `POST /api/surveys/:id/uploads`

特点：

- 允许已发布问卷的公开附件上传
- question-scoped 上传要求 `questionId + submissionToken`
- 返回 `uploadToken`
- 答卷提交阶段的最小文件引用是 `id + uploadToken`
- 未提交的临时附件会按 TTL 清理

## 12. 当前已知缺口

- 系统 smoke 已支持端口与测试库隔离，适合并行 CI
- Playwright WebServer 输出仍有 `util._extend` 与 `defineProps`/`defineEmits` 警告
- 审计、部门、职位、角色的完整后台 CRUD 仍未进入浏览器级回归
- 高级分析 / SPSS 仍主要是界面占位

## 13. 建议的开发顺序

1. 把 Playwright 继续扩到审计、部门、职位、角色 CRUD、回收站和更多权限边界。
2. 为文件、消息、审计与更多后台动作补浏览器级回归和异常路径用例。
3. 继续维护 [`.github/workflows/ci.yml`](../.github/workflows/ci.yml) 中两个 smoke 入口的 matrix 并行执行。
4. 继续把后台模块向统一的 `policy + service + repository + contract` 收口。
