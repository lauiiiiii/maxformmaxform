# TrustForm 开发指南

基于源码状态更新时间：`2026-03-29`

本文仅以当前仓库实际代码为准，适用于 `maxformmaxform` 这一套前后端工程，不再沿用旧版文档中的 MongoDB / ClickHouse 双库方案描述。

## 1. 当前技术栈

- 前端：Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router + ECharts
- 后端：Node.js 18+ + Express 4 + Knex 3 + MySQL
- 文件上传：Multer，本地磁盘存储，后端通过 `/uploads` 提供静态访问
- 鉴权：JWT
- 导出：ExcelJS + zip 附件打包
- 测试：
  - 后端：Node.js 内置 `node:test`
  - 联调：`scripts/system-smoke.mjs`
  - 前端：`vue-tsc --noEmit` + `vite build`
  - E2E：Playwright

## 3. 环境要求

- Node.js 18 或更高版本
- npm 9 或更高版本
- MySQL 8.x

建议本地保留以下端口：

- 前端开发服务：`63000`
- 后端 API：`63002`
- smoke 端口：`63102`
- MySQL：默认按 `.env.example` 使用 `3309`

## 4. 环境变量

后端读取 `backend/.env`，当前示例变量如下：

```env
DB_HOST=127.0.0.1
DB_PORT=3309
DB_USER=root
DB_PASSWORD=123456
DB_NAME=survey_system

JWT_SECRET=please-change-me
JWT_EXPIRES_IN=7d

FRONTEND_URL=http://127.0.0.1:63000
PORT=63002
UPLOAD_PENDING_TTL_HOURS=24
```

补充说明：

- 上传大小上限由 `backend/src/config/index.js` 控制，当前默认单文件 10MB。
- `UPLOAD_PENDING_TTL_HOURS` 控制未提交上传附件的清理窗口。
- 生产环境必须替换 `JWT_SECRET`。

## 10. 文件上传机制

上传相关实现位于：

- `backend/src/routes/files.js`
- `backend/src/routes/surveys.js`
- `backend/src/utils/uploadStorage.js`
- `backend/src/models/File.js`

当前分两类：

### 10.1 管理后台普通上传

- `POST /api/files/upload`
- `POST /api/files/upload/image`

要求登录。

### 10.2 问卷公开上传

- `POST /api/surveys/:id/uploads`

特点：

- 允许已发布问卷的公开附件上传
- question-scoped 上传要求 `questionId + submissionToken`
- 返回 `uploadToken`
- 答卷提交阶段的最小文件引用是 `id + uploadToken`
- 未提交的临时附件会按 TTL 清理

## 12. 当前已知缺口

- 流程与题库仓库前端页面已存在，但后端接口和前端 API 仍未落地
- 系统 smoke 仍依赖固定端口与共享测试库，不适合并行执行
- Playwright WebServer 输出仍有 `util._extend` 与 `defineProps`/`defineEmits` 警告
- 高级分析 / SPSS 仍主要是界面占位

## 13. 建议的开发顺序

1. 为流程与题库仓库补真实后端接口与前端 API。
2. 把 Playwright 覆盖面继续扩到答卷下载、消息中心、文件夹管理和更多权限边界。
3. 若要支持并行 CI，先改造 smoke 端口与测试库隔离策略。
4. 继续把后台模块向统一的 `policy + service + repository + contract` 收口。
