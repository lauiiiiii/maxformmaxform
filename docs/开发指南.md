# TrustForm信任表单 - 开发指南

## 快速开始

### 环境要求
- Node.js 18+（建议 LTS）
- 主库：MongoDB 5.0+（启用副本集以支持 Change Streams；也可通过适配器切换至 MySQL、PostgreSQL、SQL Server、SQLite、人大金仓、达梦、OceanBase 等）
- ClickHouse 23+（用于分析聚合，可用 Docker 启动）
- npm 或 yarn

### 安装和运行

#### 1. 克隆项目
```bash
git clone <项目地址>
cd 问易答-调查系统
```

#### 2. 安装依赖

**后端:**
```bash
cd backend
npm install
cp .env.example .env
# 编辑 .env 文件：设置 DB_CLIENT（默认 mongo）、主库连接串（例如 MONGODB_URI 或 MYSQL_URI）、ClickHouse（CLICKHOUSE_URL/CLICKHOUSE_DB）以及 JWT_SECRET
```

**前端:**
```bash
cd frontend
npm install
```

#### 3. 启动服务（完整版：Mongo + ClickHouse）

**启动 MongoDB（副本集，示例为单节点）：**
```powershell
# 准备数据目录（Windows 示例）
if (!(Test-Path 'C:/data/db')) { New-Item -ItemType Directory -Path 'C:/data/db' | Out-Null }

# 终端A：以副本集模式启动
mongod --dbpath C:\data\db --replSet rs0

# 终端B：初始化副本集（仅第一次）
mongosh --eval "rs.initiate()"
```

**启动后端（app.cjs，端口 63002）：**
```powershell
cd backend
$env:PORT=63002
node app.cjs
```

**启动前端:**
```bash
cd frontend
npm run dev
```

访问 http://localhost:63000 查看应用；健康检查 http://127.0.0.1:63002/health（应看到 dbClient/mongoOk/clickhouseOk 等字段）。

### 使用 Docker 启动（可选：仅演示 ClickHouse）
```bash
# 启动 ClickHouse（如果本机未安装）
docker run -d --name ch-server -p 8123:8123 -p 9000:9000 clickhouse/clickhouse-server:23
```

## 数据库适配器架构

- 所有业务数据访问统一走 `backend/src/database` 中的适配器门面；控制器通过 `database.getModel('User')` 等方式按需获取模型。
- 默认适配器为 MongoDB（Mongoose），支持 Change Streams 以驱动 ClickHouse 异步写入；未来如需新增 MySQL/PostgreSQL/SQL Server/SQLite/人大金仓/达梦/OceanBase 等，只需实现对应适配器并在 `models/index.js` 注册模型。
- 运行时通过 `.env` 的 `DB_CLIENT` 指定适配器（`mongo`、`mysql` 等），并配置对应的连接串（例如 `MONGODB_URI` 或 `MYSQL_URI`）。
- 健康检查 `/health` 会返回 `dbClient`、`dbOk` 以及 Mongo/ClickHouse 等子项状态，可用于探针与告警。

### 新增适配器的步骤
1. 在 `backend/src/database/adapters/` 下创建 `<Engine>Adapter.js`，继承 `BaseAdapter`，实现 `connect`、`disconnect`、`getModel` 等方法。
2. 在 `backend/src/database/index.js` 注册新适配器，并映射 `DB_CLIENT=<engine>`。
3. 在 `backend/src/models/index.js` 中输出复用的模型定义或提供ORM实体。
4. 更新 `.env.example`、部署文档以及健康检查提示，确保全链路可观测。

- 如需快速单库联调，可运行 `backend/simple-server.js`（内存态），但不包含多库适配逻辑，仅供前端开发调试。

## 核心功能实现

### 数据库适配层
- 通过 `database/index.js` 暴露统一的 `connect/close/getModel` 接口，业务代码不直接依赖具体数据库驱动。
- MongoDB 适配器负责加载所有 Mongoose 模型并暴露给控制器；其他数据库可按需实现实体映射。
- `answersReplicator` 订阅 Mongo Change Streams，将答卷转换为 ClickHouse JSONEachRow 以保持分析库同步。

### 1. 用户认证系统
- JWT Token 认证
- 密码加密存储 (bcrypt)
- 登录状态管理 (Pinia)
- 路由权限控制

### 2. 问卷创建流程
1. 创建问卷基本信息
2. 添加和编辑题目
3. 设置问卷配置
4. 预览和发布

### 3. 题目类型系统
- 后端权威类型为字符串枚举：`input | textarea | number | date | time | email | url | phone | radio | checkbox | rating | scale | matrix | upload | ranking | slider`
- 前端编辑器内部存在数字枚举（legacy），提交/读取时双向映射；详见 `docs/题型规范.md`
- 组件按需渲染，不同题型使用不同输入控件

### 4. 数据收集和存储
- 实时保存草稿
- 防重复提交
- 数据验证
- 文件上传处理
- 通过数据库适配层统一写入主库，必要时在适配器中实现不同数据库的事务/连接管理

### 5. 数据分析和可视化
- 提交答卷先写入 Mongo（Answer 集合）
- 后台复制器通过 Mongo Change Streams 将明细异步写入 ClickHouse（表：`trustform.survey_answers`）
- 分析接口直接从 ClickHouse 聚合：`GET /api/surveys/:id/analytics`
- 可视化：前端使用 ECharts 等渲染图表

## 数据库设计

> 当前默认实现基于 MongoDB 模式定义；若切换到其他数据库适配器，请在对应 ORM/迁移脚本中复用同一领域模型结构，并确保索引策略按目标库特性调整。

### 核心集合
1. **users** - 用户信息（`User`）
2. **surveys** - 问卷信息（`Survey`）
3. **questions** - 题目信息（`Question`）
4. **answers** - 答卷记录（`Answer`）

### 索引策略（示例）
```javascript
// Survey 集合（可根据业务增加）
db.surveys.createIndex({ creatorId: 1, createdAt: -1 })
db.surveys.createIndex({ status: 1 })

// Question 集合
db.questions.createIndex({ surveyId: 1, order: 1 })

// Answer 集合（对齐 models/Answer.js）
db.answers.createIndex({ projectId: 1, createdAt: -1 })
db.answers.createIndex({ respondentId: 1 })
db.answers.createIndex({ sessionId: 1 })
db.answers.createIndex({ ipAddress: 1, projectId: 1 })
```

## API 接口文档

### 认证相关
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/logout` - 用户登出
- `GET /api/auth/me` - 获取当前用户信息

### 问卷相关
- `GET /api/surveys` - 获取问卷列表
- `POST /api/surveys` - 创建问卷
- `GET /api/surveys/:id` - 获取问卷详情
- `PUT /api/surveys/:id` - 更新问卷
- `DELETE /api/surveys/:id` - 删除问卷
- `POST /api/surveys/:id/publish` - 发布问卷
 - `GET /api/surveys/:id/results` - 简要结果（Mongo 聚合：总数/最近）
 - `GET /api/surveys/:id/analytics` - 分析（ClickHouse 聚合）

### 题目相关
- `GET /api/surveys/:id/questions` - 获取问卷题目
- `POST /api/surveys/:id/questions` - 添加题目
- `PUT /api/questions/:id` - 更新题目
- `DELETE /api/questions/:id` - 删除题目

### 回答相关
- `POST /api/surveys/:id/responses` - 提交回答（写 Mongo；复制器异步写 CH）
- `GET /api/surveys/:id/responses` - 获取回答列表（如需，需鉴权，项目里可按需补充）

## 部署指南

### 生产环境部署

#### 1. 服务器配置
- Ubuntu 20.04+ 或 CentOS 8+
- 至少 2GB 内存
- Node.js 18+
- MongoDB 5.0+（或其他已实现的主库适配器）
- Nginx

#### 2. 部署步骤

**安装依赖:**
```bash
# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 MongoDB
sudo apt-get install mongodb

# 安装 Nginx  
sudo apt-get install nginx
```

**部署应用:**
```bash
# 克隆代码
git clone <项目地址>
cd 问易答-调查系统

# 安装依赖并构建
cd frontend && npm install && npm run build
cd ../backend && npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件（设置 DB_CLIENT、主库连接串、ClickHouse、JWT 等）

# 启动应用（后端）
npm install -g pm2
pm2 start app.cjs --name survey-backend --update-env
```

**配置 Nginx:**
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
    
  location /api {
  proxy_pass http://localhost:63002; # 完整版后端默认端口
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

#### 3. 使用 Docker 部署
如需容器化 ClickHouse，可直接使用官方镜像；应用后端与前端容器化请依据企业规范自建 Dockerfile 与 compose/Helm 模版。

## 安全措施

### 数据安全
1. **输入验证**: 严格验证用户输入
2. **注入防护**: 对 ClickHouse 查询使用参数绑定或受控 SQL 拼接，并对 path 参数做格式校验（例如 ObjectId 正则）
3. **XSS防护**: 对用户输入进行转义
4. **CSRF防护**: 使用 CSRF Token

### 接口安全
1. **身份验证**: JWT Token 验证
2. **权限控制**: 基于角色的权限控制
3. **接口限流**: 防止恶意请求
4. **HTTPS**: 生产环境使用 HTTPS

### 数据加密
1. **密码加密**: 使用 bcrypt 加密存储
2. **敏感数据**: 敏感字段单独加密
3. **传输加密**: HTTPS 传输加密
