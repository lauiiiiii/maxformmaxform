## 4. 额外观察

### 4.1 并行 smoke 补充复测

在完成 smoke 端口与测试库隔离改造后，今天补做并行实验：

- `node scripts/system-smoke.mjs`
- `cmd /c npm run smoke:system`

并行执行结果：

- 两个入口均通过，`64 / 64`
- 两个实例拿到不同的实际监听端口
- 两个实例使用不同的临时测试库，并在结束后自动清理

结论：

- `scripts/system-smoke.mjs` 未显式设置 `PORT` 时会让后端监听 `0` 端口，由系统分配空闲端口
- smoke 默认基于 `DB_NAME` 为每次运行创建独立测试库，避免并发相互污染
- `SMOKE_SHARED_DB=1` 可切回共享库模式，`SMOKE_KEEP_DB=1` 可保留临时库排障

### 4.2 Playwright WebServer 仍有警告

浏览器回归虽然通过，但启动日志里仍出现：

- `util._extend` 废弃警告
- `defineProps` / `defineEmits` 编译器宏导入警告

这些不影响本轮通过结论，但值得后续清理。

## 7. 建议

1. 保持 smoke 的端口与测试库隔离策略，并在 CI 中继续以并行 job 运行两个入口。
2. 基于流程、题库仓库、文件管理已落地的真实接口，继续补后台能力清单和 API/E2E 文档。
3. 将 Playwright 继续扩到审计、部门、职位、角色 CRUD、回收站与更多权限边界场景。
