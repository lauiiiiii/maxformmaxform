## 4. 本轮系统冒烟更新

本轮对 `scripts/system-smoke.mjs` 做了两类修正。

### 4.1 稳定性修正

- `runId` 从秒级时间戳改为 `Date.now() + pid + random`，避免两个入口并行触发时用户名/编码冲突
- 预热逻辑从已失效的 `seed()` 调用改为 `ensureBaseRoles()`，对齐当前 `backend/src/db/seed.js` 的真实导出
- 文件清理改为可跳过已在批量删答卷流程中删除的附件，避免重复清理导致误报
- 审计日志断言改为匹配当前分页返回契约，消除脚本对旧数组结构的误判

### 4.2 覆盖面补充

新增联调断言：

- 问卷列表读取
- 答卷计数接口
- 答卷详情接口
- 批量删答卷
- 删答卷后统计与计数同步
- 问卷关闭接口
- 关闭后禁止继续提交

因此系统冒烟口径从上一轮的 `56 / 56` 扩展到 `64 / 64`。

## 6. 风险与缺口

虽然本轮系统验证通过，但仍存在以下未完全覆盖范围：

- 浏览器级 E2E 已落地，但目前只覆盖编辑器、公开填写、结果页 3 条主链路
- 结果快照命中/重建路径主要靠后端单测验证，系统冒烟只覆盖结果接口正向行为
- 用户、角色、部门、文件等后台模块尚未形成与问卷域同等强度的服务化集成验证
- 系统冒烟脚本依赖固定端口与共享测试库，不适合并行执行多个实例

## 7. 建议

建议后续按以下顺序补强：

1. 为编辑器、填写页、结果页补一层浏览器级回归。
2. 把浏览器 E2E 从 3 条主链路扩展到管理后台、上传题、回收站与权限场景。
3. 给结果快照补命中率与重建耗时观察口径。
4. 继续把后台管理域的路由层逻辑收缩到 service/repository。
