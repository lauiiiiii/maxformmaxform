# 前端问卷编辑器：解耦重构步骤（循序渐进）

> 目标：把“上帝组件”CreateSurveyPage.vue 拆成清晰的类型/工具/状态与组件分层；每一步可独立提交、可回退，保持功能与行为不变。

## 总体策略
- 顺序：先抽“纯类型与纯函数”，再抽“状态/动作”，最后抽“UI 组件”。
- 键：所有映射使用 question.id，彻底摆脱索引错位。
- 单一来源：将“问卷编辑状态”集中在 store/composable 管理，组件只展示+发事件。
- 规则下沉：协议映射/解析/校验/工具函数统一挪到 utils/mappers。
- 弹窗与工具条：独立组件，props+emit 明确契约，便于复用与测试。

## 关键契约示例

### BatchAddDialog.vue
- props: `visible: boolean`, `defaultType: number`
- emits:
  - `confirm(list: Array<{ type: number; title: string; options?: string[] }>)`
  - `cancel()`

### SurveyQuestionItem.vue
- props: `question: Question`, `index: number`, `active: boolean`, `areAllNumbersHidden: boolean`
- emits: `edit(index)`, `moveUp(index)`, `moveDown(index)`, `duplicate(index)`, `delete(index)`, `change(question)`

### useSurveyEditorStore（片段）
- state:
  - `questions: Question[]`
  - `editingIndex: number`
  - `areAllNumbersHidden: boolean`
  - `titleRichMap: Record<string, { rich: boolean; html: string }>`
- actions:
  - `addQuestionByType(type)`
  - `addQuestionAfter(index)`
  - `moveQuestionUp/Down(index)`
  - `duplicateQuestion(index)`
  - `deleteQuestion(index)` // 清理 `titleRichMap[id]`
  - `toggleNumbers(nextState?)`
  - `applyTitleRteContent(id, html)`

## 迁移顺序与回归检查清单
- 新增/复制/删除/移动题目是否正常
- “隐藏题号” → 新增题 → 新题默认是否跟随
- 标题富文本：编辑保存、删除题、再新增题，是否串内容
- 批量添加：是否跟随隐藏题号策略；解析是否正确
- 预览/填写页：`titleHtml` 安全渲染是否稳定

## 质量门槛
- Build：Vite Dev 可运行；TypeScript 编译通过（逐步修正历史告警）
- Lint：ESLint 如开启则通过
- 关键交互：手测 PASS

## 建议的提交流程
- 每个步骤作为独立 commit/PR：描述变更范围与“行为不变”说明
- commit 内包含：
  - 更新的文件列表与目的
  - 简单手测要点（如上清单）
- 若某步风险较高（例如状态集中化），可先引入 store 与页面并行运行一小段时间，再切换引用。
