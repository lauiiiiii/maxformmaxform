# TrustForm：为私有化而生的新一代问卷系统

> 2026-03-28 注：本文档中的“MongoDB + ClickHouse 双库”表述属于历史方案，和当前仓库主线实现不一致。当前系统结构请先参考 `README0326.md`、`目录速览.md` 与 `docs/系统结构评估与优化建议-2026-03-27.md`。

把话放在前面——这不是“又一个”问卷系统，这是“新范式”。

> 技术宣言：我们重构开源问卷系统的技术基准——
> - 非 Java 技术栈：后端 Node.js + 前端 Vue 3 + TypeScript + Vite（现代、轻快、可运维）
> - 减法设计：做对 80% 企业刚需的最短路径，而非把 100 个开关“全塞满”
> - 真·双数据库：MongoDB 做主存，ClickHouse 做分析；写入与分析天然解耦
> - 多数据库适配：基于适配器模式，可切换 MySQL / PostgreSQL / SQL Server / SQLite / 人大金仓 / 达梦 / OceanBase 等
> - 为独立部署而生：Nginx + PM2 一把梭，/health 即刻可观测，文档即落地
> - 安全优先：JWT + RBAC、CORS 白名单、最小权限、审计日志、受控 SQL 拼接

## 我们与众不同的 8 个理由

1) 新一代技术栈（现代全栈）
- 前端：Vue 3 + TypeScript + Vite，快、轻、易维护
- 后端：Node.js + Express，天然契合云原生与容器化
- 工程即文档：目录清晰、注释充分、上手快

2) 极简而强大（减法设计）
- 专注“创建-发布-收集-分析”闭环，不被冗余功能拖慢交付
- 更少的配置、直觉的交互、可演进的扩展点

3) 真正的双数据库架构 + 多数据库适配（他们单库，我们双库 + 可插拔）
- MongoDB 做“主存”（系统事实来源）
- ClickHouse 做“分析”（高吞吐聚合）
- 通过 Mongo Change Streams 异步复制到 ClickHouse：主链路不阻塞、分析更丝滑
- 底层使用数据库适配器模式：默认 MongoDB，也可扩展 MySQL、PostgreSQL、SQL Server、SQLite、人大金仓、达梦、OceanBase 等企业数据库

4) 为独立部署而生（私有化友好）
- 支持内网/离线环境，Nginx + PM2 一把梭
- 健康检查 /health，开箱可观测
- 文档齐全：环境安装、开发指南、企业部署规范、API 清单

5) 安全优先（默认安全）
- JWT 鉴权、基于角色的访问控制（RBAC）
- CORS 白名单、HTTPS（生产）、最小权限 DB 账号
- 审计日志与可追溯性（接口/操作日志）
- 防注入：ClickHouse 查询做受控拼接 + 参数校验

6) 面向分析的工程化
- 分析接口直连 ClickHouse，前端图表即刻出图
- 宽表建模与可扩展索引策略，满足多维统计
- 支持历史回灌脚本（Answer → ClickHouse）

7) 易于运维与扩展
- /health 就绪信号：dbClient / dbOk / mongoOk / clickhouseOk 一目了然
- 配置即环境变量：PORT、DB_CLIENT、MONGODB_URI、CLICKHOUSE_URL/DB 等
- 容量规划建议与备份指引，工程化友好

8) 真实可落地
- 不是“玩具 Demo”，而是面向“企业内网可交付”的实用工程
- 从团队协作到上线部署，提供完整落地链路
- 数据库适配器让选型自由：MongoDB 起步，企业可无缝接入自有数据库栈

## 与传统方案的差异化

| 维度 | 传统开源（多为 Java 单库） | TrustForm（Node + Vue 双库） |
| --- | --- | --- |
| 架构 | 单数据库，分析与写入耦合 | MongoDB（主存）+ ClickHouse（分析），异步复制 |
| 复杂度 | 功能堆叠、学习曲线陡峭 | 减法设计，路径清晰、上手快 |
| 前端 | 旧技术栈，迭代慢 | Vue 3 + TS + Vite，现代工程 |
| 部署 | 运维复杂 | 私有化友好，Nginx + PM2 快速上线 |
| 可观测 | 缺乏统一就绪信号 | /health 返回 mongoOk/clickhouseOk |
| 分析性能 | 受限于主库 | ClickHouse 天然擅长聚合与大数据扫描 |

> 他们把“功能做满”，我们把“体验做顺”。他们追求“万能”，我们定义“好用 + 可运维”的新标准。

```
[前端 Vue3] ⇆ [/api 网关/Nginx] ⇆ [后端 app.cjs]
                               ├── MongoDB（主存，副本集，Change Streams）

数据流：提交答卷 → 写 Mongo → 复制器监听 Insert → 转写 ClickHouse → 分析接口查 CH
```
