
# 问卷题型规范（现行实现版）

本规范与当前代码保持一致：
- 后端的题型是“字符串枚举”，写入 MongoDB（Question.type）
- 前端编辑器内部有一套“数字编码（legacy）”，用于 UI 选择与排序，提交/读取时与后端字符串类型做双向映射

建议：以“后端字符串枚举”为唯一权威类型，前端的数字编码仅作为编辑器内部表示，未来可逐步收敛到字符串。

## 一、后端题型（权威）

来源：`backend/src/models/Question.js`

允许的字符串类型（enum）：
- input（单行文本）
- textarea（多行文本）
- number（数字）
- date（日期）
- time（时间）
- email（邮箱）
- url（网址）
- phone（电话）
- radio（单选）
- checkbox（多选）
- rating（评分）
- scale（量表）
- matrix（矩阵）
- upload（文件上传）
- ranking（排序）
- slider（滑块）

题目数据结构（精简自 Mongoose 模型）：

```jsonc
{
  "type": "radio|checkbox|input|...",   // 字符串枚举（后端权威）
  "title": "题目标题",
  "description": "题目说明",
  "required": false,
  "order": 1,
  "options": [                            // 仅选项题
    { "label": "选项1", "value": "1", "order": 1,
      "rich": false, "desc": "", "exclusive": false,
      "defaultSelected": false, "hidden": false,
      "visibleWhen": /* Mixed：选项显示逻辑（OR-AND）*/ null }
  ],
  "optionOrder": "none|all|flip|firstFixed|lastFixed", // 选项顺序策略
  "optionGroups": [ { "name":"分组名", "from":1, "to":5, "random":false } ],
  "groupOrderRandom": false,
  "autoSelectOnAppear": false,
  "logic": { "visibleWhen": /* Mixed：题目显示逻辑（OR-AND）*/ null },
  "jumpLogic": /* Mixed：跳题逻辑（按题号或end/invalid）*/ null,
  "matrix": { "rows": [{"label":"","value":""}], "columns": [{"label":"","value":""}] },
  "rating": { "min":1, "max":5, "step":1, "labels": {"min":"","max":""} },
  "upload": { "accept":"", "maxSize":5000000, "multiple":false }
}
```

答案数据结构（来源：`backend/src/models/Answer.js` 中 `answers[]` 单元）：

```jsonc
{
  "questionId": "ObjectId",         // 指向后端 Question._id
  "questionType": "radio|input|...", // 冗余类型（便于查询）
  "value": /* Mixed：字符串/数组/数值… */,
  "text": "可选文本",
  "files": ["url1", "url2"]
}
```

提交接口契约（POST /api/surveys/:id/responses）：
- 前端提交的 `answers[].questionId` 为题号的“1-based 序号（字符串或数字）”，后端会按题目 `order` 建立映射，转换为真实的 `Question._id`。例如第1题提交 `questionId: 1`。

## 二、前端题型（编辑器内部数字编码，legacy）

来源：
- 基础常量：`frontend/src/constants/questionTypes.ts`（1~13）
- 创建页本地扩展：`frontend/src/views/survey/CreateSurveyPage.vue`（包含 14~42 的扩展占位）

已在全局常量中的数字编码（1~13）与常用含义：

| 数字编码 | 名称         | 默认映射到后端字符串 |
|----------|--------------|----------------------|
| 1        | 填空题       | input                |
| 2        | 简答题       | textarea             |
| 3        | 单选题       | radio                |
| 4        | 多选题       | checkbox             |
| 5        | 量表题       | scale（待接入渲染）   |
| 6        | 矩阵题       | matrix（待细化）      |
| 7        | 下拉题       | 当前映射为 radio     |
| 8        | 滑动条题     | slider               |
| 9        | 多项填空题   | 暂未接入（建议走 input 多项） |
| 10       | 矩阵填空     | 暂未接入              |
| 11       | 排序题       | ranking              |
| 12       | 比重题       | 暂未接入（建议自定义）|
| 13       | 文件上传题   | upload               |

在 CreateSurveyPage.vue 中的本地扩展（14~42）多为“UI 占位/规划”，尚未全部接入持久化；其中已做映射的包括：
- 14（日期）→ 后端 `date`
- 8（滑动条）→ 后端 `slider`

映射逻辑（片段）：

```ts
// 提交前（前端 -> 后端）
function mapLegacyTypeToServer(t: number): string {
  switch (t) {
    case 1: return 'input'
    case 2: return 'textarea'
    case 3: return 'radio'
    case 4: return 'checkbox'
    case 7: return 'radio'   // 下拉题当前以单选持久化
    case 11: return 'ranking'
    case 13: return 'upload'
    case 14: return 'date'
    case 8: return 'slider'
    default: return 'input'
  }
}

// 读取后（后端 -> 前端）
function mapServerTypeToLegacy(t: string): number {
  switch (t) {
    case 'input': return 1
    case 'textarea': return 2
    case 'radio': return 3
    case 'checkbox': return 4
    case 'ranking': return 11
    case 'upload': return 13
    case 'date': return 14
    case 'slider': return 8
    default: return 1
  }
}
```

提示：未来建议在前端也直接使用后端字符串类型，减少双向映射与“占位类型”的歧义。

## 三、题目与逻辑配置

- 选项顺序策略：`optionOrder = none | all | flip | firstFixed | lastFixed`
- 选项分组与随机：`optionGroups[]`、`groupOrderRandom`
- 选项级显示逻辑：`options[i].visibleWhen = Cond[][]`（OR-AND 结构）
- 题目级显示逻辑：`logic.visibleWhen = Cond[][]`
- 跳题逻辑：`jumpLogic`（支持 byOption、目标题号字符串、'end'、'invalid' 等）

示例（单选题，带选项扩展与逻辑）：

```jsonc
{
  "type": "radio",
  "title": "您对我们的服务满意吗？",
  "required": true,
  "order": 1,
  "options": [
    { "label": "非常满意", "value": "1", "order": 1, "defaultSelected": true },
    { "label": "满意",       "value": "2", "order": 2 },
    { "label": "一般",       "value": "3", "order": 3 },
    { "label": "不满意",     "value": "4", "order": 4 }
  ],
  "optionOrder": "none"
}
```

## 四、提交与统计

- 提交时：`answers[].questionId = 题号(1-based)`，由后端映射为 `Question._id`
- 后端保存后会将 `questionType` 冗余在答案中，便于统计
- 统计分析：当前将答案明细异步写入 ClickHouse 的 `survey_answers` 表，用于高性能聚合

## 五、如何新增题型（建议流程）

1) 后端：将新字符串类型加入 `Question.js` 的 enum；必要时扩展题型专属字段结构
2) 前端：
   - 编辑器 UI：新增卡片与渲染；尽量直接使用“后端字符串类型”
   - 若暂用数字编码：补充前后端映射（mapLegacyTypeToServer / mapServerTypeToLegacy）
3) 提交/读取：确保 toServerPayload 与 getSurveyRaw 的映射正确
4) 答案校验：根据题型在后端新增最小校验（如必填、选项数量等）
5) 文档：在本文件追加该题型的说明、数据结构与示例
