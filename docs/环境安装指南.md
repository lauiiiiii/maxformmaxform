# 🚀 TrustForm信任表单 - 环境安装指南

> 2026-03-28 注：本文档仍残留旧版 MongoDB / ClickHouse / `app.cjs` 安装说明，尚未完全切换到当前 `Express + Knex + MySQL` 实现。当前本地开发与联调请优先参考 `README0326.md`、`目录速览.md` 与 `backend/README.md`。

## 📋 系统要求

### 最低配置要求
- **操作系统**: Windows 10+ / macOS 10.15+ / Ubuntu 18.04+
- **内存**: 4GB RAM (推荐 8GB+)
- **存储**: 2GB 可用空间
- **网络**: 稳定的互联网连接

## 🛠️ 环境安装步骤

### 1. 安装 Node.js

#### Windows 用户

**方法一：官网下载（推荐）**
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 **LTS 版本**（长期支持版，推荐 18.x 或 20.x）
3. 运行下载的 `.msi` 安装文件
4. 按照安装向导完成安装（建议保持默认设置）
5. 重启命令提示符或 PowerShell

**方法二：使用 Chocolatey**
```powershell
# 以管理员身份运行 PowerShell
Set-ExecutionPolicy Bypass -Scope Process -Force
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# 安装 Node.js
choco install nodejs
```

**方法三：使用 Winget**
```powershell
# Windows 10/11 内置包管理器
winget install OpenJS.NodeJS
```

#### macOS 用户

**方法一：官网下载**
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 LTS 版本的 `.pkg` 文件
3. 双击安装

**方法二：使用 Homebrew（推荐）**
```bash
# 安装 Homebrew（如果未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Node.js
brew install node
```

**方法三：使用 nvm（版本管理）**
```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重启终端后安装 Node.js
nvm install --lts
nvm use --lts
```

#### Linux (Ubuntu/Debian) 用户

**方法一：使用 NodeSource 仓库（推荐）**
```bash
# 更新包列表
sudo apt update

# 安装 curl
sudo apt install -y curl

# 添加 NodeSource 仓库
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -

# 安装 Node.js
sudo apt-get install -y nodejs
```

**方法二：使用 Snap**
```bash
sudo snap install node --classic
```

**方法三：使用 nvm**
```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重启终端
source ~/.bashrc

# 安装最新 LTS 版本
nvm install --lts
nvm use --lts
```

### 2. 验证安装

安装完成后，打开命令提示符（Windows）或终端（macOS/Linux），运行以下命令验证：

```bash
# 检查 Node.js 版本
node --version
# 应该显示类似：v18.17.0 或 v20.5.0

# 检查 npm 版本
npm --version
# 应该显示类似：9.6.7
```

### 3. 配置 npm（可选但推荐）

```bash
# 设置 npm 镜像源（加速下载）
npm config set registry https://registry.npmmirror.com

# 或者使用 yarn（可选）
npm install -g yarn
```

### 4. 安装 MongoDB（本地开发）

#### Windows
1. 访问 [MongoDB 官网](https://www.mongodb.com/try/download/community)
2. 下载 MongoDB Community Server
3. 运行安装程序，选择 "Complete" 安装
4. 安装 MongoDB Compass（图形界面工具）

#### macOS
```bash
# 使用 Homebrew
brew tap mongodb/brew
brew install mongodb-community
```

#### Linux (Ubuntu)
```bash
# 导入 MongoDB 公钥
wget -qO - https://www.mongodb.org/static/pgp/server-6.0.asc | sudo apt-key add -

# 添加 MongoDB 仓库
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/6.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-6.0.list

# 更新包列表并安装
sudo apt-get update
sudo apt-get install -y mongodb-org
```

### 5. 启动 MongoDB

#### Windows
- MongoDB 通常会作为 Windows 服务自动启动
- 或手动启动：在服务管理器中找到 MongoDB 服务并启动

#### macOS/Linux
```bash
# 启动 MongoDB 服务
sudo systemctl start mongod

# 设置开机自启动
sudo systemctl enable mongod

# 检查状态
sudo systemctl status mongod
```

## 🧱 双数据库说明（MongoDB + ClickHouse）

本项目采用“业务在 MongoDB，分析在 ClickHouse”的双库结构：
- 提交答卷先写 MongoDB（Answer 集合）
- 通过 MongoDB Change Streams 异步复制“答卷明细”到 ClickHouse（表：trustform.survey_answers）
- 分析接口（如 /api/surveys/:id/analytics）直接查询 ClickHouse 做聚合

要让异步复制生效，MongoDB 必须以“副本集模式”运行；ClickHouse 需可达（默认 http://127.0.0.1:8123）。

## � 数据库适配器模式

- 所有后端模块通过 `backend/src/database` 暴露的适配器访问数据库：`const database = require('../database')`，再调用 `database.getModel('User')`、`database.getOrm()` 等。
- 默认适配器为 MongoDB（基于 Mongoose），由 `.env` 中的 `DB_CLIENT=mongodb` 控制；你可以新增适配器以接入 MySQL、PostgreSQL、SQL Server、SQLite、人大金仓、达梦、OceanBase 等数据库。
- 适配器负责 `connect()` / `disconnect()` / `isConnected()` / `getModel()`，并在 `/health` 中提供 `dbClient`、`dbOk`、`mongoOk`、`clickhouseOk` 等状态指标，便于跨平台部署和可观测。
- 所有脚本与路由（如 `app.cjs`、`app.js`、`initAdmin.js`、`src/controllers/*`）都已经改为通过适配器取模型，避免直接依赖某种数据库实现。

### 1) 启动 MongoDB（副本集模式）

如果你本地还没有副本集配置，可用单节点副本集快速初始化（演示用）：

```powershell
# 确保数据目录存在（示例：C:\data\db）
if (!(Test-Path 'C:/data/db')) { New-Item -ItemType Directory -Path 'C:/data/db' | Out-Null }

# 终端A：以副本集模式启动 MongoDB
mongod --dbpath C:\data\db --replSet rs0

# 终端B：初始化副本集（仅第一次）
mongosh --eval "rs.initiate()"
```

如果已有 Mongo 服务，请将服务启动参数加入 `--replSet rs0` 并执行一次 `rs.initiate()`。

### 2) 启动 ClickHouse（可选：Docker 快速运行）

### 3) 启动后端（完整版，端口 63002）

### 5) 快速自测

2. 在前端填写并提交一份答卷

## 🔧 常见问题解决

### Node.js 安装问题

**问题 1**: 提示 "node 不是内部或外部命令"
```bash
# 解决方案：重启命令提示符，或手动添加到环境变量
# Windows: 添加 C:\Program Files\nodejs 到 PATH
# 验证安装：
where node  # Windows
which node  # macOS/Linux
```

**问题 2**: npm 安装速度慢
```bash
# 使用国内镜像源
npm config set registry https://registry.npmmirror.com
# 或使用 cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
```

**问题 3**: 权限错误（macOS/Linux）
```bash
# 修改 npm 全局安装目录权限
sudo chown -R $(whoami) $(npm config get prefix)/{lib/node_modules,bin,share}
```

### MongoDB 连接问题

**问题 1**: 连接被拒绝
```bash
# 检查 MongoDB 是否正在运行
sudo systemctl status mongod  # Linux
brew services list | grep mongodb  # macOS

# 启动 MongoDB
sudo systemctl start mongod  # Linux
brew services start mongodb/brew/mongodb-community  # macOS
```

**问题 2**: 端口冲突
- 默认端口 27017 被占用
- 修改 `backend/.env` 文件中的 `MONGODB_URI`

### 项目启动问题

**问题 1**: 端口被占用
```bash
# 查找占用端口的进程
netstat -ano | findstr :63000  # Windows
lsof -i :63000  # macOS/Linux

# 杀死进程
taskkill /PID <进程ID> /F  # Windows
kill -9 <进程ID>  # macOS/Linux
```

**问题 2**: 依赖安装失败
```bash
# 清理缓存重新安装
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

## 🧪 常见问题（双库相关）

1) /health 显示 clickhouseOk=false
- 说明 ClickHouse 未启动或 8123 不可达，请检查服务或端口映射。

2) 提交后 analytics 没有数据
- 多半是 Mongo 没有以副本集模式运行，Change Streams 未生效；请按上文设置 `--replSet` 并 `rs.initiate()`。
- 也可能是刚提交尚未同步，等待 1~2 秒再试。

3) 依然想快速联调而不装数据库？
- 可运行“简化内存版”（simple-server.js），根目录 `后端-启动.cmd` 默认以 63002 启动纯内存服务，但不具备持久化与 ClickHouse 统计。
