# PR 摘要 - 2026-03-28 服务化重构与浏览器回归落地

## 变更主题

本轮变更相对 `2026-03-26` 的核心变化有三件事：

1. 把问卷系统从“题型增强阶段”推进到“服务化分层 + 共享契约”阶段。
2. 把问卷编辑器从巨型页面推进到“页面壳 + 子面板 + composable”结构。
3. 把浏览器级 Playwright 回归从空白入口推进到真实用例，并同步刷新测试与文档口径。

## 主要改动

### 1. 后端服务化与共享契约扩展

- 问卷路由收缩为薄路由，核心逻辑拆入：
  - `surveyQueryService`
  - `surveyCommandService`
  - `surveyUploadService`
  - `surveyResultsService`
- 答卷链路拆成：
  - `answerQueryService`
  - `answerCommandService`
  - `answerExportService`
- 管理域增加：
  - `deptService`
  - `roleService`
  - `positionService`
  - `folderService`
  - `messageService`
  - `auditService`
  - `userService`
- 新增仓储层：
  - `deptRepository`
  - `roleRepository`
  - `positionRepository`
  - `folderRepository`
  - `messageRepository`
  - `auditRepository`
  - `userRepository`
  - `surveyAggregateRepository`
- 新增共享契约：
  - `shared/management.contract.js`
  - `shared/pagination.contract.js`
  - `shared/survey.contract.js`
- 新增 HTTP/事务基础设施：
  - `backend/src/http/errors.js`
  - `backend/src/http/asyncRoute.js`
  - `backend/src/http/downloadResponse.js`
  - `backend/src/db/transaction.js`

影响：

- 路由层职责显著收缩。
- 前后端列表返回、DTO、错误码口径更统一。
- 管理域开始摆脱大量 `route -> model` 直连写法。

### 2. 结果统计加入快照与观测字段

- 新增 `SurveyResultsSnapshot` 模型。
- `surveyResultsService.js` 增加：
  - snapshot hit / miss / rebuild 逻辑
  - rebuild duration 记录
  - request duration / sample tier / payload baseline
  - read error / persist error 统计
- 结果接口继续复用共享题目统计模型，保持前后端统计口径一致。

影响：

- 结果接口不再完全依赖每次实时重算。
- 系统具备最小结果缓存与过程观测能力。

### 3. 编辑器与填写页结构拆分

- 编辑器新增页面壳与子面板：
  - `CreateSurveyPageShell.vue`
  - `CreateSurveyQuestionConfigPanel.vue`
  - `CreateSurveyQuestionListPanel.vue`
  - `CreateSurveyLogicSettings.vue`
  - `CreateSurveyPublishPanel.vue`
- 编辑器新增领域 composable：
  - `editor-core.ts`
  - `editor-logic.ts`
  - `editor-batch.ts`
  - `editor-quota.ts`
  - `editor-richtext.ts`
- `CreateSurveyPage.vue` 收缩为薄入口。
- 填写页新增 hook：
  - `useFillSurveyVisibility.ts`
  - `useFillSurveyJumpLogic.ts`
  - `useFillSurveyQuota.ts`
  - `useFillSurveyUpload.ts`

影响：

- 编辑器告别单页巨石。
- 新能力可以落到更清晰的领域边界，而不是继续堆在页面组件里。

### 4. Playwright 浏览器回归正式落地

- 新增 `frontend/playwright.config.ts`
- 新增 `frontend/e2e/survey-pages.spec.ts`
- 新增 `backend/scripts/start-e2e.js`
- `frontend/package.json` 接入 `npm run test:e2e`

当前用例覆盖：

- 编辑器创建问卷、添加题目、发布
- 公开填写页提交流程
- 结果页摘要和分析视图

影响：

- `2026-03-26` 中的 `Error: No tests found` 问题已消失。
- 浏览器回归第一次进入可执行状态。

### 5. 冒烟与运行脚本刷新

- 后端新增：
  - `db-migrate.js`
  - `db-seed.js`
  - `start-dev.js`
  - `start-prod.js`
- 后端启动不再隐式执行迁移和种子，改为显式脚本控制。
- `scripts/system-smoke.mjs`：
  - `runId` 改为毫秒级 + pid + 随机后缀
  - 新增问卷列表、答卷计数、答卷详情、批量删答卷、关闭后禁提等断言
  - 修复审计日志分页断言
  - 修复附件清理重复删除问题

影响：

- 系统冒烟从 `56 / 56` 扩展到 `64 / 64`。
- 并发入口下的 `runId` 冲突风险已消除。

### 6. 文档整体刷新

- 新增：
  - `docs/测试报告-2026-03-28.md`
  - `docs/系统结构评估与优化建议-2026-03-27.md`

影响：

- 文档口径已对齐当前源码，而不是继续停留在 `2026-03-26` 之前的状态。

## 影响判断

- 正向影响
  - 问卷主链路从题型增强推进到结构化分层
  - 管理域开始获得统一契约和服务边界
  - 编辑器和填写页的维护成本开始下降
  - 浏览器回归已经不再缺位
  - 结果统计具备快照与最小可观测字段

- 残余风险
  - 管理域的分层一致性仍未完全追平问卷域
  - 编辑器上下文字段仍然偏多
  - 浏览器回归只覆盖 3 条主链路
  - 系统冒烟依赖固定端口与共享测试库，不适合并行执行
  - 地域统计仍依赖来源地字段稳定落库

## 建议 follow-up

1. 继续把管理域剩余路由逻辑向 `service / repository` 收口。
2. 扩大 Playwright 覆盖面到管理后台、上传题、权限边界和回收站场景。
3. 继续缩减 `editorContext` 暴露面，收紧页面壳与子面板之间的契约。
4. 为结果快照补更完整的命中率和重建耗时观察指标。
