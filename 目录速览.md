# 目录速览

本文档只按当前仓库源码描述，不沿用旧的 MongoDB / ClickHouse / `app.cjs` / `simple-server.js` 方案。

基于源码状态更新时间：`2026-03-29`

## 一句话理解

- `backend/`：`Express + Knex + MySQL` 后端，问卷域分层最完整。
- `frontend/`：`Vue 3 + TypeScript + Vite` 单页应用，已覆盖问卷编辑、填写、结果和后台管理。
- `shared/`：前后端共享题型注册表、题目统计模型和部分管理域 DTO。

## 根目录

- `backend/`：后端服务。
- `frontend/`：前端工程。
- `shared/`：题型与管理契约共享层。
- `scripts/`：系统冒烟、清理等辅助脚本。
- `docs/`：项目文档与阶段性评估。
- `README0326.md`：项目总览与启动说明。
- `目录速览.md`：仓库结构速览。

## backend/

当前后端真实入口和结构如下：

- `server.js`：实际启动入口，先检查 MySQL 连通性，再启动 HTTP 服务。
- `app.js`：Express 应用装配层，挂载安全中间件、静态资源、路由和错误处理。
- `initAdmin.js`：初始化管理员账号。
- `scripts/start-dev.js`、`scripts/start-prod.js`：开发/生产启动包装脚本。
- `src/config/`：环境变量与运行配置。
- `src/db/`：Knex 连接、迁移、种子、事务封装。
- `src/models/`：面向表的访问层。
- `src/routes/`：HTTP 路由入口。
- `src/policies/`：访问控制与问卷策略判断。
- `src/services/`：问卷、认证、后台管理业务服务。
- `src/repositories/`：聚合写入仓储与后台数据访问仓储。
- `src/middlewares/`：鉴权与错误处理中间件。
- `src/utils/`：题型校验、上传存储等工具。
- `test/`：Node `node:test` 路由与服务测试。

### 后端当前调用链

问卷域当前更接近下面这条链路：

```text
HTTP Request
  -> route
    -> policy / access check
      -> service
        -> repository + transaction
          -> model
            -> JSON response
```

管理域与认证当前更接近：

```text
HTTP Request
  -> route
    -> authRequired
      -> service
        -> actor/admin policy
          -> repository / model
            -> DTO contract
              -> JSON response
```

说明：

- `src/routes/surveys.js` 已主要承担参数提取和响应封装。
- `src/services/authService.js` 统一负责注册、登录、当前会话读取。
- `src/services/userService.js`、`roleService.js`、`deptService.js`、`positionService.js`、`folderService.js`、`messageService.js`、`auditService.js` 已承接后台路由逻辑。
- 路由层不再总是显式写死 `requireRole('admin')`，部分接口的管理员判断已经下沉到 service/policy。

## frontend/

前端当前真实结构如下：

- `src/main.ts`、`src/App.vue`：应用入口。
- `src/router/index.ts`：路由定义与前置守卫。
- `src/layouts/`：后台布局壳。
- `src/stores/`：Pinia 状态管理。
- `src/api/`：接口封装。
- `src/views/admin/`：后台管理页面。
- `src/views/survey/`：问卷创建、编辑、填写、结果、答卷管理。
- `src/composables/`：跨页面组合式逻辑。
- `src/composables/editor/`：问卷编辑器子 composable。
- `src/components/`：通用组件。
- `src/mappers/`、`src/types/`：前端数据映射与类型定义。
- `src/utils/`：题型注册、导入、图表、上传等工具。
- `src/styles/`：全局样式与设计令牌。
- `e2e/`：Playwright 浏览器级回归。

### 问卷编辑器当前调用链

```text
route
  -> CreateSurveyPage / FillSurveyPage
    -> CreateSurveyPageShell / page components
      -> useSurveyEditor / useFillSurvey*
        -> composables/editor/*
        -> api/*
          -> backend /api/*
```

说明：

- `CreateSurveyPage.vue` 已是薄入口。
- `useSurveyEditor.ts` 现为装配层，聚合 `editor-core`、`editor-logic`、`editor-batch`、`editor-quota`、`editor-richtext`。
- 填写页逻辑已拆成 `useFillSurveyVisibility.ts`、`useFillSurveyJumpLogic.ts`、`useFillSurveyQuota.ts`、`useFillSurveyUpload.ts`。
- `src/views/admin/Flows.vue` 与 `src/views/admin/Repos.vue` 已接入共享 DTO，但 `src/api/flows.ts`、`src/api/repos.ts` 仍为占位实现。

## shared/

- `shared/questionTypeRegistry.js`：题型定义、legacy `uiType`、渲染类型、统计类型、提交语义。
- `shared/questionModel.js`：题目级统计聚合逻辑。
- `shared/management.contract.js`：用户、角色、部门、职位、文件夹、消息、审计，以及流程/题库仓库 DTO 与查询归一化。

这是当前问卷题型语义、结果统计口径和部分管理域响应格式的统一入口。

## 运行与联调

- 后端默认端口：`63002`
- 前端默认端口：`63000`
- smoke 端口：`63102`
- 健康检查：`GET /health`

前端开发代理：

- `/api` -> `http://127.0.0.1:63002`
- `/uploads` -> `http://127.0.0.1:63002`

注意：

- 当前启动后端不会自动执行 `migrate()` 或 `seed()`。
- 首次启动前应手动执行：
  - `cd backend`
  - `npm run db:migrate`
  - `npm run db:seed:dev`
- `scripts/system-smoke.mjs` 与 `frontend` 下的 `npm run smoke:system` 不能并行运行，否则会争用 `63102`。
