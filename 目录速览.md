# 目录速览

快速了解这个仓库的结构、端口与每个目录的作用，帮助你高效上手。

## 前后端各自做啥（一句话版）
- 后端（backend）：提供 API、认证鉴权、数据持久化（MongoDB）与分析聚合（ClickHouse）。完整版默认端口 63002（`app.cjs`）。
- 前端（frontend）：提供页面 UI、路由与交互，通过 `/api` 代理调用后端，开发端口 63000。

## 根目录
- backend/：后端服务（Node.js + Express）
- frontend/：前端工程（Vue 3 + Vite + TypeScript）
- docs/：文档（环境安装、开发指南、企业部署规范、题型规范等）
- scripts/ 与清理脚本：`clean.bat`、`scripts/cleanup.ps1`（清理 node_modules、dist 等可再生目录）
- 后端/前端一键脚本：`后端-启动.cmd`、`前端-启动.cmd`（按需使用）

## backend/（后端）
- app.cjs：完整后端入口（推荐）。包含 Mongo 连接、ClickHouse 工具、异步复制器、路由与 `/health` 健康检查；默认端口 63002（可用 `PORT` 覆盖）。
- simple-server.js：极简内存版（本地联调/演示），不依赖数据库。
- app.js：ESM 版本的服务骨架（可选）。
- server-cjs.js：CJS 简化启动（备用）。
- initAdmin.js：初始化管理员账号脚本。
- package.json / package-simple.json：依赖与脚本定义。
- .env.example：后端环境变量示例（包含 Mongo、ClickHouse、JWT 等）。
- src/
  - constants/：常量与枚举（如 `questionTypes.js`）。
  - controllers/：接口控制器（业务处理）。
  - middlewares/：中间件（如 `auth.js`）。
  - models/：数据模型（`Survey`/`Question`/`Answer`/`User` 等）。
  - routes/：路由定义（`auth.(js|cjs)`、`surveys.(js|cjs)`）。
  - utils/：工具与封装（`clickhouse.js`、`exporter.js`、`zipper.js`）。
  - workers/：后台作业（`answersReplicator.js`，负责 Mongo→ClickHouse 异步复制）。

端口说明：
- 完整版后端默认 63002，可通过环境变量 `PORT` 覆盖。
- 简化内存版可按 VS Code 任务使用 3002/63002/3004 等端口。

## frontend/（前端）
- index.html：入口 HTML
- src/
  - main.ts、App.vue：应用入口与根组件
  - router/：路由配置（`router/index.ts`）
  - views/：页面视图（如 Dashboard、问卷管理/填写/分析）
  - components/：通用组件（富文本弹窗等）
  - composables/：组合式函数（如 `useAuth`）
  - api/：接口请求封装
  - services/：领域服务（与后端接口语义化对接）
  - store/：全局状态
  - styles/：全局样式与设计令牌（`design-tokens.css`）
  - assets/：静态资源
  - constants/：前端常量与枚举（题型等）
  - utils/：工具函数
- vite.config.ts：开发服务器与代理配置（默认前端 63000，`/api` 代理到后端 63002）。

## 运行提示（简要）
- 推荐使用 VS Code 任务：
  - 后端（完整版）：`run:backend-63002`
  - 或后端（简化内存版）：`run:backend-simple-63002` / `run:backend-simple`（3002） / `run:backend-simple-3004`
  - 前端：`run:frontend-63000`
- 健康检查：`http://127.0.0.1:63002/health`（应显示 `mongoOk`/`clickhouseOk`）。
- 分析接口：`GET /api/surveys/:id/analytics`（直接查询 ClickHouse）。
- 完整版后端依赖：MongoDB（副本集）与 ClickHouse。简化版不依赖数据库，仅用于本地演示。

更多细节与环境准备请参考：`docs/环境安装指南.md` 与 `docs/开发指南.md`。
